What is Clay?

Clay is a Javascript Workflow based on GULP, Command-Line-Tool and a Static Server.

Prebaked Receipts
Command Line Tool
Cloud Static Server

Clay Receipts ( Javascript Workflow ) automate the development of your application, use any framework and Gulp plug-in completely adjustable to your preference.

CLAY Command Line Tool to publish previews and final versions of your project to the web to a customizable URL: APP.3votapp.com. Where APP is the name of your App. ie: docs.3votapp.com.

Preview and Final Versions are hosted in Amazon AWS Static Servers - configured to hold the most demanin loads.

What is CLAY for Salesforce.com

Clay for Salesforce is an extension to build Web and Mobile Apps that run inside Salesforce and Salesforce1. It's used to build businesss Apps.

CLAY features a Development Server that redefines how visualforce is coded and makes it 10X Faster. It also automates deployment of Visualforce Pages and Static Resources which represent another huge performance driver.

With CLAY developers can build Javascript Apps traditionaly using Salesforce Data via Visualforce Pages and Visualforce Remoting.

Why should I use CLAY?

If your startup or department has an automated workflow for building front end apps, and runs the infrastructure in a first level cloud - where every project follows a standard workflow and Apps can be deployed and updated any second by pressing a button - then you don't need CLAY.

Otherwise - you either need to develop CLAY yourself or you can jump in and start using CLAY right now.

On the other hand if you are ok by not having an automated frontend development workflow that publishes to static cloud servers - getting starting with CLAY today is smart. Your operation will probably get more complicated and you'll need CLAY soon.

Installation

Clay is an Node Package Module and used NODE.JS to run on Mac, Linux and Windows Computers.

The first step is to install NODE.JS which is a really easy with it's installers.

Install CLAY with NPM

After NODE.JS is installed go to your Terminal ( CMD in Windows ) and install Clay CLI using the command on the right. Clay is installed globally. You may need to use SUDO on unix. ie: sudo npm install...

Clay Receipts

3VOT provide several receipts which are pre-configured development environments.

There is nothing special about them and you can personalize them or build your own if you are an advanced developer - instructions down the fold.

Start by downloading or cloning them from Github.

Setup of Receipts

Once you have downloaded/cloned a receipt - go to Terminal or CMD in Windows and navigate to your project root folder.

From this root folder - you'll create, develop and publish apps.

Install Dependencies

CLAY Receipts uses NPM Dependencies - which is great for frontend projects. Download and Installation is automated by NPM ( Node Package Manager )

Required dependencies are kept in Package.json - you can search over 100K in npmjs.org

.ENV

Clay projects require a .env file on the root folder of the project.

Make sure you create a .env file that looks like this. This file contains secret keys, make sure is mentioned in .gitignore file so it's not uploaded to your repository.

Follows Convention over Configuration.

  • -- htdocs html, fonts and static files
  • -- images images

The following folders don't need to be modified

  • - build auto-generated for local server
  • - dist auto-generated for deployment
  • - gulp Gulp Plugins - auto loaded

Salesforce Receipts

Clay4Salesforce Receipts are a very similar to Open Web Receipts - they share folder structure and Gulp Automation Tasks.

The difference between Salesforce Receipts lies in the fact the we don't use HTML pages - but instead we use Visualforce Pages. The File src/htdocs/index.html is actually a Visualforce Page

When SF Receipts run in Development Mode ( explained in The CLI Section ) , files like CSS, JS and Images point to the local files. When SF Receipts are published to Salesforce they point to Static Resources.

In order for this to work you must use the {{3vot}} tag before each file, look at the example on the right. {{3VOT}} tags get's replaced for http://localhost:3000/xxxxxx in Development and StaticResource Tag.in Production

Feel free to use any Visualforce Tags. You can use a HEAD tag - but don't use a BODY tag - since it's not supported by Visualforce

Salesforce .env File

The .env file when used with Salesforce Receipts must also include the Admin Username, Password and Security Token.

This credentials are used to manage Visualforce Pages and Static Resources, make sure .env files are keeped in the .gitignore file so they are not committed to your repo.

Clay Command Line Tool

The Command Line is the Developers most important tool when developing apps. CLAY CLI makes is really easy to publish and preview apps online.

Create App

Every app needs to be created, this process checks if the name is available and also sets the name of the App in package.json. Your url will be http://APP.3votapp.com.

Development

In order to start working on your project, make sure your location in Terminal ( CMD in windows ) is the the root of the project and start the server.

Make changes to your HTML,CSS or Javascripr and Clay will reload the app on your browser using BrowserSync Live Reloader.

Preview

Use Clay smart Previews Feature in order to share the current status of your project. The project it's uploaded to a staging URL you can just share with internal stakeholders.

Publish

Build the Production Version of your Project and Deploys it to 3VOT's Static Server that can hold massive amounts of visitors.

Clay for Salesforce Users

Simply add the salesforce keywork to each command ( dev and publish )

ie: "clay salesforce dev" and "clay salesforce publish"
